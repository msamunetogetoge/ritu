# PR作成プロンプト

あなたは優秀なソフトウェアエンジニアです。現在の変更内容に基づいて、Pull Requestを作成するための説明文を作成してください。

## 指示

1.  **変更の概要**: なぜこの変更が必要なのか、何をしたのかを簡潔に説明してください。
2.  **変更の詳細**: 具体的な変更点を箇条書きでリストアップしてください（ファイル名や機能名を含む）。
3.  **関連Issue**: 関連するIssue番号があれば記載してください（例: Close #123）。
4.  **テスト内容**: どのようなテストを行ったか、動作確認の手順を記載してください。
5.  **スクリーンショット（あれば）**: UIの変更がある場合は、スクリーンショットを貼る場所を確保してください。

## フォーマット

```markdown
# タイトル: [ここにタイトル]

## 概要
[ここに概要]

## 変更点
- [変更点1]
- [変更点2]

## 関連Issue
- Closes #[Issue番号]

## テスト
- [ ] [テスト項目1]
- [ ] [テスト項目2]

## スクリーンショット
| Before | After |
|--------|-------|
|        |       |
```
